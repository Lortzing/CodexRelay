# CodexRelay signing and notarization

The Release workflow supports:

- Windows Authenticode signing for `cxr.exe`, the Setup executable, and the Inno Setup uninstaller.
- macOS Developer ID signing for the PyInstaller executable, Developer ID Installer signing for the PKG, and Apple notarization/stapling for both PKG and DMG.
- Explicitly suffixed `-unsigned` artifacts when no signing credentials are configured.

Signing certificates and private keys must be obtained and controlled by the repository owner. They require personal or company identity verification and cannot be generated by the repository or by a third party on your behalf.

## Windows Authenticode

### Obtain a certificate

Use a publicly trusted code-signing CA or cloud-signing service. Modern publicly trusted certificates commonly keep private keys in a hardware token, HSM, or cloud service.

The included workflow supports an exportable PFX. If your provider only supports a hardware token or remote signing API, replace the SignTool invocation in `scripts/build_windows_installer.ps1` with the provider client, such as Azure Trusted Signing, DigiCert KeyLocker, or SSL.com eSigner.

A self-signed certificate is suitable only for internal testing and does not remove the unknown-publisher warning for normal users.

### Configure GitHub Secrets

Encode an exportable PFX as one-line Base64.

PowerShell:

```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("codesign.pfx")) | Set-Clipboard
```

macOS:

```bash
base64 -i codesign.pfx | tr -d '\n' | pbcopy
```

Linux:

```bash
base64 -w 0 codesign.pfx
```

Add these repository Actions secrets:

| Secret | Value |
|---|---|
| `WINDOWS_CERTIFICATE_PFX_BASE64` | One-line Base64 PFX |
| `WINDOWS_CERTIFICATE_PASSWORD` | PFX password |

The workflow imports the certificate temporarily, signs with SHA-256 and an RFC 3161 timestamp, verifies the signatures, and removes the certificate from the runner.

Verify locally:

```powershell
Get-AuthenticodeSignature .\cxr.exe | Format-List
Get-AuthenticodeSignature .\CodexRelay-Setup-*.exe | Format-List
signtool verify /pa /v .\CodexRelay-Setup-*.exe
```

## macOS Developer ID and notarization

### Obtain certificates

An active Apple Developer Program membership is required. Create both certificate types:

- `Developer ID Application` for `cxr` and DMG signing.
- `Developer ID Installer` for PKG signing.

Export each certificate together with its private key from Keychain Access as a password-protected P12:

```text
DeveloperIDApplication.p12
DeveloperIDInstaller.p12
```

Find the exact identities:

```bash
security find-identity -v -p codesigning
security find-identity -v
```

Typical names:

```text
Developer ID Application: Your Name (TEAMID)
Developer ID Installer: Your Name (TEAMID)
```

Create an Apple ID app-specific password for `notarytool` and record your Team ID.

### Configure GitHub Secrets

Configure all nine values. Partial configuration fails deliberately so that a partially signed or non-notarized file is not mistaken for a finished release.

| Secret | Value |
|---|---|
| `MACOS_APPLICATION_CERT_P12_BASE64` | Application P12 Base64 |
| `MACOS_APPLICATION_CERT_PASSWORD` | Application P12 password |
| `MACOS_INSTALLER_CERT_P12_BASE64` | Installer P12 Base64 |
| `MACOS_INSTALLER_CERT_PASSWORD` | Installer P12 password |
| `MACOS_APPLICATION_IDENTITY` | Full Developer ID Application identity |
| `MACOS_INSTALLER_IDENTITY` | Full Developer ID Installer identity |
| `APPLE_ID` | Apple ID email |
| `APPLE_TEAM_ID` | Apple Team ID |
| `APPLE_APP_PASSWORD` | App-specific password |

The workflow creates a temporary keychain, imports both identities, signs the PyInstaller one-file executable with hardened runtime, signs and notarizes the PKG, creates and signs the DMG, notarizes it, staples both artifacts, and deletes the temporary keychain.

Verify locally:

```bash
codesign --verify --verbose=2 --strict cxr
pkgutil --check-signature CodexRelay.pkg
spctl --assess --type install --verbose=4 CodexRelay.pkg
xcrun stapler validate CodexRelay.dmg
spctl --assess --type open --verbose=4 CodexRelay.dmg
```

## Secret handling

- Never commit PFX, P12, passwords, Base64 certificate data, or private keys.
- Consider placing the release job behind a protected GitHub Environment with required reviewers.
- Do not expose signing secrets to pull requests from forks.
- Rotate app-specific passwords and revoke compromised certificates immediately.

## Unsigned artifacts

With no signing secrets, the workflow still creates testable assets with explicit names:

```text
CodexRelay-Setup-<version>-windows-<arch>-unsigned.exe
CodexRelay-Portable-<version>-windows-<arch>-unsigned.zip
CodexRelay-<version>-macOS-<arch>-unsigned.dmg
```

These builds are appropriate for internal testing, not as the primary download for nontechnical users.
